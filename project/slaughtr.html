The Game

Slaughtr is a fun prototype made during the Week of Awesome III (gamedev.net game jam) for the theme "Death is Useful". We were only 3 programmers and no artists to make that prototype.

You play as Death, and the goal of the prototype is to kill as many annoying Slaughtr users (ideally everyone) without people being aware that an arbitrary force is killing their friends.

Even though it's technically a game, I keep calling Slaughtr a prototype because we were trying a lot of different things, and you can't really win or lose. For example, the simulation of the people's behavior took multiple variables (like the number of friends, post frequency, random variables) that were very hard to balance correctly together.

What I Worked On

Tools used:

  • Unity
  • Visual Studio C#


Some of the features I implemented:

  • Placing the pictures as equidistant from each other as possible using the golden spiral technique
  • Rotation of the sphere
  • Smoothly lerping to a picture when the user clicks on it
  • Names, pictures and comments generators
  • The interface on the right that's illustrating the state of the simulation in real time
  • The action of killing someone
